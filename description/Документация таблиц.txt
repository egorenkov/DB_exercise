База данных предназначена для хранения информации о космических объектах и их запусках. Состоит из двух нормализованных таблиц, связанных между собой через ключевые поля.

## Таблица: `launches` (Запуски)

Хранит информацию о фактах запусков космических объектов.

### Структура таблицы

| Поле | Тип данных | Обязательное | Описание |
|------|------------|---------------|-----------|
| `launch_id` | INTEGER | PRIMARY KEY | Уникальный идентификатор запуска |
| `position` | TEXT | ✓ | Позиция или орбитальная позиция объекта |
| `status` | TEXT | ✓ | Текущий статус объекта (активный, неактивный и т.д.) |
| `launch_place` | TEXT | ✓ | Место запуска (космодром) |
| `launch_data` | TEXT | ✓ | Дата запуска объекта |

### Ключевые поля
- **Первичный ключ**: `launch_id`
---

## Таблица: `objects` (Объекты)

Содержит информацию о конкретных космических объектах (спутниках, аппаратах).

### Структура таблицы

| Поле | Тип данных | Обязательное | Описание |
|------|------------|---------------|-----------|
| `object_id` | INTEGER | PRIMARY KEY | Уникальный идентификатор объекта |
| `launch_id` | INTEGER | ✓ | Внешний ключ к таблице launches |
| `name` | TEXT | ✓ | Название космического объекта |
| `model` | TEXT | ✓ | Модель или тип объекта |
| `operator` | TEXT | ✓ | Оператор/владелец объекта |


### Ключевые поля
- **Первичный ключ**: `object_id`
- **Внешний ключ**: `launch_id` → `launches(launch_id)`
- **Индексы**: рекомендуется создать индекс на `operator` и `model`

---

## Связи между таблицами

```
launches (1) ←→ (1) objects
    ↑                    ↑
 launch_id           launch_id (FK)
```

- Каждой записи в таблице `launches` соответствует одна запись в таблице `objects`
- Связь "один к одному" через поле `launch_id`
